git init - начало работы с гитом
git add . - добавить все
git commit -m "комментарий" - закоммитить
git status - статус
git log --oneline - контрольные точки
git show - показать что менялось
git checkout idxxxxx - перейти к контрольной точке
git checkout master - вернуться к основному
git revert id  - отмена коммита
git reset id --hard - удаление коммита и откат изменений (хард)
git rm - отмена действий
git rm -cached "file" - отменить из статуса ожидания

Игнор
Работает функция .gitignore очень просто:

Создаем вручную файл под названием .gitignore и сохраняем его в директорию проекта.
Внутри файла перечисляем названия файлов/папок, которые нужно игнорировать, каждое с новой строки.
Файл .gitignore добавляем, коммитим и отправляем на сервер, как любой другой файл в проекте. Пример исключения перечисленных выше файлов выглядит так:
*.log
build/
node_modules/
.idea/
my_notes.txt
